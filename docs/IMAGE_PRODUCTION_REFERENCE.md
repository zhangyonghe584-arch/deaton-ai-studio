# Deaton Auto image production reference

## Brand position

Deaton Auto presents professional remote vehicle programming and technical support for repair shops, technicians, locksmiths, and specialist repair professionals. Images should demonstrate technical competence, remote-programming capability, and a credible solution to a real vehicle problem—not equipment sales or generic repair advertising.

## Case story

A finished image set should make this progression clear:

1. Vehicle and reported issue
2. Diagnosis or technical evidence
3. Remote programming procedure
4. Verified result

The source images and supplied case information are the facts. Do not invent faults, diagnostic outcomes, programming steps, customer statements, or repaired functions.

## Source-image handling

- Prioritize the subject, orientation, clarity, information relevance, and safe crop of each supplied image.
- Do not crop away an essential vehicle detail, change evidence, fabricate a result, or add unrelated source material.
- Empty source slots are acceptable; the local generator must communicate the missing image rather than prevent a draft.

## Visual direction

Use a clear professional automotive-technology direction: restrained, trustworthy, technically focused, and readable. Avoid racing imagery, low-cost advertisement styling, excessive effects, and artificial-looking imagery.

The application UI uses warm white and light grey surfaces, graphite text, deep blue actions, and a small orange warning/accent color. Generated images use the same restrained family while retaining high contrast for publishing.

## Output

The MVP produces six 1080 × 1920 PNGs for portrait social publishing. The logo, title hierarchy, case facts, technical sequence, and final result must remain readable at phone scale.

## AI boundary

AI may summarize selected real images, propose copy, suggest a storyboard, and suggest local-layout parameters. The user reviews and confirms the editable plan before it informs local generation.
